Troubleshooting a Java application

  1. Debugger
  2. Log/print statements
  3. JMX
  4. Adhoc interactive mechanism
How many people in the room use Java day to day? You might recognise some of the debugging strategies that I have listed here. (pause) Just quick comment regarding their limitations: To use the debugger you generally need to have a development environment and be able to narrow the problem down to a series of breakpoints. If you're investigating a race condition the Debugger can change the nature of the problem your trying to study. Log/print statements need to be planned before compilation, can slow down the application and can be hard to read. JMX or equivalent, need to be designed into the application to be useful. We will see that JVM-breakglass does not suffer from the same limitations. (time 1min 30 sec : total time 3 mins)

What is JVM-breakglass?

  1. Open Source Maven Library
  2. Integrated on any JVM process
  3. Console onto a JVM process
JVM-breakglass is a Java library (click), easy to integrate (click) which gives access to a REPL console into a JVM process. (time 1 : total time 5 mins)

What are the main JVM-breakglass features?

  1. Interactive prompt
  2. See inside private members
  3. Call arbitrary methods
  4. Creation of new object instances
  5. Creation of new classes
  6. Monitor object state
  7. No need to use Clojure to develop the application
Java developers are used to the "write the code", compile, package, test and deploy cycle. This library provides a console access onto your JVM process (), so you can display the content of any object (), call methods (), create objects at runtime () without having to plan for it in advance or go through the same slow dev cycle everytime we run a small experiment? (time 1 : total time 5 mins)

How does it work?

How does it work? (pause) This is your java application with it's data structures and processing classes. JVM-breakglass starts an nRepl server listening on a port your choice. At any moment an nRepl client i.e. lein for example can connect onto it to inspect the JVM. (time 1 min ; total 6 mins)
