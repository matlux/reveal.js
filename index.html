Troubleshooting a Java application

  1. Debugger
  2. Log/print statements
  3. JMX
  4. Adhoc interactive mechanism
How many people in the room use Java day to day? What do you normally do to track a bug? (pause) Most of you must have used the debugger or print statements which are well established ways of tracking bugs in many languages (inc java). The debugger is mostly useful when the bug can be reproduced on a development machine and can be narrowed down to a series of breakpoints. Debuggers have to stop the world which can change the nature of the problem your trying to study. Log/print statements can be very powerful but need to be planned before compilation, add clutter to the code and can slow down the application. Some of you might have used JMX or implemented an interactive mechanism (say via rest calls),very good to retreive or send selected information to your remote objects. Good to deal with a dynamic and complex problem. However needs a good deal of scaffolding and design needs to be applied to provide a powerful investigation tool. (time 1min 30 sec : total time 3 mins)

What can JVM-breakglass provide?

  1. Interactive prompt
  2. See inside private members
  3. Call arbitraty methods
  4. Creation of new object instances
  5. Monitor object state
  6. Overwrite class definition on the fly
  7. No need to use Clojure to develop the application
In this crowd, we're probably all used to the fact that most function languages are equipped with a REPL which enable to poke around a live process to figure out what's going on? Why can't we leverage this concept with any JVM application? (pause) Java developers are used to the "write the code", compile, package and deploy cycle. Why can't we prompt the process to display the content of an object, call methods, create objects at runtime without having to plan for it in advance or go through the same slow dev cycle everytime to run a small experiment? With this tool you can do that and more... like creating new objects and monitor object state without having to specifically use Clojure in your project. (time x : total time x mins)

How does it work?

This is your java application with it's data structures and processing classes. boundary of the jvm. JVM-breakglass with it's helper functions. JVM-breakglass starts an nRepl server listening on a port your choice. At any moment an nRepl client i.e. lein for example can connect onto it to inspect the JVM.
